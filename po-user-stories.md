You are a senior product manager and agile expert. I will provide you with documents.

Your task is to analyze all documents and generate a comprehensive set of user stories for the project.

### Instructions:

* Carefully review and synthesize information from all three documents.
* Identify key features, functionalities, and system behaviors.
* Break them down into clear, actionable user stories.

### Output Requirements:

For each user story, provide:

* **Title**
* **User Story** (in the format: *As a [user type], I want [goal], so that [benefit]*)
* **Description** (concise but informative)
* **Acceptance Criteria** (in bullet points, using testable conditions)
* **Priority** (High / Medium / Low)
* **Dependencies** (if any)
* **Notes/Assumptions** (if applicable)

### Additional Guidelines:

* Group user stories by **epics or features**.
* Ensure stories are **independent, negotiable, valuable, estimable, small, and testable (INVEST principles)**.
* Avoid duplications across documents.
* Resolve conflicts between documents by prioritizing PRD intent, then HLD, then Implementation details.
* Include both **functional and non-functional** requirements (e.g., performance, security, scalability).
* Where details are missing, make reasonable assumptions and clearly state them.
* Highlight any ambiguities or gaps in a separate section.

### Output Format:

Organize your response as:

1. **Overview of Features/Epics**
2. **User Stories (Grouped by Epic)**
3. **Non-Functional User Stories**
4. **Assumptions & Clarifications Needed**
5. **Potential Risks or Gaps**
